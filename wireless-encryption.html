üìâ Problem Description

Wireless networks transmit data using electromagnetic waves. This means:

  • Data is broadcasted in a shared medium, anyone nearby can potentially intercept it.
  • Wireless environments are significantly less secure than wired ones by default.

If the wireless traffic is not encrypted (e.g., open Wi-Fi or weak WEP), attackers can:

  • Packet Sniffing: Capture unencrypted traffic from the air, even without joining the network.
  • Session Hijacking: Steal active session cookies and impersonate users (e.g., Facebook login).
  • Evil Twin: Set up a fake AP with the same SSID to lure users into connecting to it.

üß™ Scenario: Sniffing + Evil Twin at Comvis Coffee

A user connects to the public Wi-Fi at Comvis Coffee, which is open or uses weak WEP encryption.

  • An attacker sets up a fake AP with the same SSID.
  • The user connects to the rogue AP unknowingly.
  • The user opens an insecure app or HTTP website.
  • The attacker intercepts session cookies or login data.

This man-in-the-middle access allows eavesdropping, credential theft, or content injection.
