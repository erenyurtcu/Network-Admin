üí£ Problem Description: DDoS & DNS Flood Attacks

Distributed Denial of Service (DDoS) attacks aim to overwhelm network resources by sending massive traffic to targets. One prevalent variant is the DNS Flood, where attackers send a deluge of DNS requests to overburden name servers or use open resolvers for reflection attacks.

Primary Objectives of Attackers:

  • Exhaust network bandwidth
  • Crash or slow down DNS resolution services
  • Disrupt application connectivity via name resolution failure
  • Consume CPU and memory of edge devices (routers/firewalls)

‚ùó Real-world behavior: Even if backend services are intact, unresolved DNS queries render the entire infrastructure unreachable. Users perceive it as "internet down."

These attacks are often difficult to trace due to:

  • Source IP spoofing (hard to block based on IP)
  • Use of legitimate DNS query formats
  • Amplification tactics via public resolvers (e.g., 1 small query ‚Üí 50x bigger response)

Defending against DNS-based DDoS requires Layer 3‚Äì7 mitigation techniques integrated across infrastructure and cloud. Suggested actions include:

  • Rate Limiting & QoS: Prioritize DNS/HTTPS traffic with policies that limit bursts and protect application traffic.
  • DNS Monitoring: Use NetFlow, sFlow, or DPI tools to flag excessive DNS traffic from unknown sources.
  • Secondary Routers: Place inline filtering routers between ISP and internal network for early detection and mitigation.
  • Edge Firewall ACLs: Deny suspicious UDP/53 queries that don‚Äôt match allowed internal patterns.
  • DNS Response Rate Limiting (RRL): Prevent reflection via authoritative DNS servers.
  • Cloud DDoS Protection: Use third-party mitigation (e.g., Cloudflare, AWS Shield, Akamai) to absorb upstream floods.
